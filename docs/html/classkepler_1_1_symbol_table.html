Detailed Description

The SymbolTable is used to map identifiers to their values.

A SymbolTable can have a parent SymbolTable, which is used to look up identifiers which are not defined in the current SymbolTable.

void kepler::SymbolTable::remove ( const String &  id,
bool  locally_only = false 
)

If the current SymbolTable contains the id, then the value is deleted and freed from memory. If 'locally_only' is set to false and the id was not found in the current SymbolTable, the parent will be checked and the value will be removed from there.

No exceptions are thrown if the id is not found.

Parameters
idThe id to remove.
